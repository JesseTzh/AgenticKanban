# 登录页工作流展示动画设计

## 目标

将 `stitch_agentickanban/code.html` 中的 AgenticKanban 工作流动画移植到登录页左侧空白区域。展示应保留参考实现的完整视觉叙事：阶段轮转、任务处理进度、AI 扫描、人工确认和任务跨阶段移动。

实现不引入额外依赖，优先复用现有 shadcn `Card` 组件和主题变量。所有新增页面元素都必须包含 `data-test-id`。

## 范围

### 包含

- 在桌面端登录页左侧展示五阶段工作流动画。
- 阶段包括需求澄清、技术拆解、开发执行、代码复核和测试验收。
- 每次循环依次执行 AI 扫描、人工确认、任务跨阶段移动和聚焦卡片轮转。
- 日间和夜间模式使用现有主题系统提供的 CSS 变量。
- 当系统启用 `prefers-reduced-motion: reduce` 时展示静态看板。
- 补充渲染、主题兼容和减少动态效果降级相关测试。

### 不包含

- 不修改登录 API、认证流程或表单交互。
- 不在平板和手机端展示工作流动画。
- 不引入第三方动画库、图标库或新字体。
- 不将动画扩展到登录页以外的页面。

## 方案选择

### 采用：React 状态机配合 CSS 动画

新增登录页专用展示组件。React 状态负责推进工作流阶段和任务状态，CSS 负责过渡、扫描、辉光、确认和背景效果。

选择该方案的原因：

- 能完整表达参考动画中的多阶段时序。
- 遵循 React 生命周期，便于清理计时器并避免挂载后的手工 DOM 修改。
- 可测试静态结构、主题兼容和减少动态效果降级。
- 不需要新增依赖。

### 未采用：直接移植 DOM 脚本

原始 HTML 使用 `document.createElement`、`innerHTML` 和手工样式更新。该方式移植速度较快，但会绕过 React 渲染模型，增加计时器清理和状态同步风险。

### 未采用：纯 CSS 循环

纯 CSS 可以实现轻量装饰动画，但难以清晰表达任务经过 AI 处理、人工确认后跨阶段流转的完整过程。

## 组件设计

新增 `web/src/features/auth/login-workflow-showcase.tsx`。

组件由以下部分组成：

- 展示区域：提供网格背景、环境辉光和裁剪边界。
- 轮播容器：提供 3D 透视，展示五张阶段卡片。
- 阶段卡片：基于 shadcn `Card`，包含阶段图标、中文副标题、英文标题和任务列表。
- 展示任务：包含任务编号、状态图标、进度条、AI Agent 指示器和扫描数据流。

阶段卡片根据与当前阶段的相对位置使用五种布局状态：

- 当前阶段：居中放大并增强描边和阴影。
- 前后相邻阶段：向两侧平移、缩小并降低透明度。
- 前后第二阶段：继续向两侧移动、缩小并进一步降低透明度。

图标使用项目现有的 `lucide-react`，不复用参考 HTML 中依赖 Material Symbols 字体的图标名称。

## 动画状态流

组件内维护五个阶段、各阶段任务列表、当前聚焦阶段和正在处理的任务状态。一个完整循环约六秒：

1. 当前阶段末尾任务进入 AI 扫描状态。
2. 扫描高光横向扫过任务卡片，进度条填充至 `100%`，AI Agent 指示器和数据流显示。
3. 扫描结束后，任务进入人工确认状态，描边短暂变为成功色，图标切换为确认状态。
4. 任务从当前阶段向右淡出，并以相反方向进入下一阶段。
5. 聚焦卡片切换到下一阶段。
6. 当工作流回到第一阶段时，按容量补充新的展示任务。
7. 任务通过最后阶段后淡出，避免任务列表无限增长。

计时器在组件卸载时必须清理。展示数据是纯前端示例数据，不触发 API 请求。

## 主题适配

动画颜色优先复用现有 CSS 变量：

- 页面背景：`--background`
- 卡片和玻璃效果：`--card`、`--glass`
- 文本：`--foreground`、`--muted-foreground`
- 描边：`--outline`
- 主要强调色：`--primary`
- 次级强调色：`--secondary-foreground`
- 成功确认色：组件局部定义稳定的绿色变量

在 `web/src/index.css` 中新增登录页展示专用样式。日间模式使用较浅网格、低透明度辉光和清晰阴影；夜间模式增强青色辉光但保持文字对比度。样式只作用于登录页展示组件，避免影响后台页面。

## 响应式和无障碍

- 登录页维持当前 `lg` 双栏布局。
- 小于 `lg` 时不渲染展示组件，只显示登录表单。
- 展示区域标记为装饰性内容，不干扰屏幕阅读器读取登录表单。
- `prefers-reduced-motion: reduce` 下不启动自动推进计时器，并禁用扫描、旋转、辉光和位移动画。左侧保留静态工作流看板。

## 登录页集成

`web/src/features/auth/login-page.tsx` 中原有 `login-empty-region` 替换为桌面端展示容器，并挂载 `LoginWorkflowShowcase`。右侧登录卡片、主题切换和认证逻辑保持不变。

## 测试

在现有 Vitest 和 Testing Library 测试体系中补充覆盖：

- 登录页包含新的工作流展示区域。
- 五个阶段卡片均渲染且包含稳定的 `data-test-id`。
- 新增展示结构不会改变用户名、密码、登录按钮和主题切换按钮。
- 减少动态效果启用时组件保持静态展示，不启动工作流推进。
- 主题切换后展示组件仍存在，并依赖主题变量而非硬编码单一暗色背景。

构建验证继续运行 `npm run test` 和 `npm run build`。
